1. Custom Power Commands (Aliases)

These commands are built into ~/.bashrc for rapid administration.

toolbox # Runs ds.sh: The health dashboard for all 10 services.
refresh # Stops, cleans orphans, and rebuilds the entire Docker stack.
fleet # Quick view of all running containers and their uptime.
logs [name] # Tail logs for a specific container (e.g., logs borg-ui-client).

Manual Repair Commands

# Force kill process on a port (if a container fails to start)
sudo lsof -i :8086
sudo kill -9 [PID]

# Reload Nginx without downtime
sudo nginx -t && sudo systemctl reload nginx

4. Nginx Logic: Sub-path Assets Fix

All apps run behind /subpath/. To prevent 404s on CSS/JS files, Nginx uses sub_filter to inject the path into HTML on-the-fly.

# Example: Borg UI Asset Fix
location /borgui/ {
  proxy_pass http://127.0.0.1:8086/;
  proxy_set_header X-Forwarded-Prefix /borgui;
  sub_filter 'href="/assets/' 'href="/borgui/assets/';
  sub_filter 'src="/assets/' 'src="/borgui/assets/';
  sub_filter_once off;
}

5. Borg Backup & Recovery Protocol

The backup system uses an SSH-enabled repository server and a Web UI.

# EMERGENCY RECOVERY STEPS
# 1. Install Docker & Nginx on fresh server.
# 2. Extract galactic_recovery.tar.gz to ~/docker_apps.
# 3. Copy nginx_configs to /etc/nginx/sites-available/.
# 4. Run 'docker compose up -d' in ~/docker_apps.

# RESET BORG ADMIN PASSWORD (If locked out)
docker compose stop borg-ui
docker compose rm -fv borg-ui # Deletes local DB
docker compose up -d

6. Cockpit Sub-path Logic

To run Cockpit inside the /cockpit/ path, the system config must be aware of its root.

# /etc/cockpit/cockpit.conf
[WebService]
Origins = https://galacticbacon.nic.edu
ProtocolHeader = X-Forwarded-Proto
UrlRoot = /cockpit

7. VS Code Persistence & Home Access

VS Code is containerized but has a "window" into the host filesystem via Docker volume binds.

# docker-compose.yml snippet
code-server:
  volumes:
    - /home/damurphy:/home/damurphy # Grants access to host home
    - /home/damurphy/.config/code-server:/config # Persists VS Code settings
